## 工作方式记录

### 1. 色卡驱动（chart variants）

**问题**：真实扫描内容 mostly 灰色调，chromatic bias 不容易观察。

**方案**：用 `examples/gen_chart_ffcs.rs` 生成合成色卡（64 colors × 4 mirrored positions），每色出现在 4 个不同 quadrant 位置，平均掉 lens/vignette 局部偏差。

**工具**：`examples/chart_analyze.rs` 按 64 色 × 4 position 采样 tile 中心 50% 区域，报每色 RMS + 按 R/G/B/Y/C/M/Gray 分组统计。诊断 flag：
- `--no-icc`：跳过 ICC transform
- `--no-usm`：跳过 USM
- `--bw-old-icc` / `--bw-bt601-g22` / `--bw-manual-g22`：BW 路径 A/B 对比
- `--icc-rel` / `--icc-sat` / `--icc-abs` / `--icc-bpc`：ICC intent 对比

**价值**：快速定位 pipeline 哪一层出问题。T53 就是色卡数据暴露 lightness 高 L 档位 FAIL 引出的。

### 2. 批量变体 + 真实扫描验证

**工具**：`examples/gen_variant_ffcs.rs` 从 `test1_raw.fff` 基础上替换 XML，生成 177 个 v_* variants。用户在 FlexColor 批量导出 TIF 作为 ground truth。

**关键坑（feedback memory 已记录）**：
- 每子目录 ≤20 文件（FlexColor 打开大目录崩溃）
- 需按 tone/color/adv 三类分 9 子目录

**价值**：chart 上发现的 STRICT/WARN 在真实扫描下是否成立。今天这个验证揭示：色卡的 WARN band（edge USM artifacts）在真实场景下基本都是 PASS，大部分"色卡 FAIL"是 chart-synthetic 特有的硬边现象。

### 3. Ghidra agent 子任务

本 session 用到 3 个 ghidra 深度调研：
- **T47 agent**：找 `FUN_702d90b0` pipeline 拓扑 + BT.601 `FUN_7025b1f0` RGB→Gray 公式
- **T53 agent**：找 `FUN_702d4720` Lightness apply 的 luma 计算（确认 `(R+G+B)/3` 权重常量 `0.333f×3 @ 0x70733f80`）
- **T56 agent**：CGammaNegCurve 是否有 film_type=2 分支（结论：无，gamma 默认普世 0.2）

每次 ghidra agent 任务输入结构：
1. 具体数值现象（像素值 + 期望 vs 实际）
2. 已尝试的假设（含失败的）
3. 候选地址列表 + 已知结论
4. 期望的 deliverable 格式

这种结构让 agent 快速定位，避免大海捞针。

### 4. systematic-debugging skill

T54 测试"BW 输出需 Hasselblad Gray kTRC gamma 2.199 编码"假设时用了 systematic-debugging 流程：
- Phase 1: 数值验证（pure B 输入 226 vs ref 239）
- Phase 2: 对比参考实现
- Phase 3: 形成单一假设
- Phase 4: 最小实现 + 测试

结果假设失败（加 gamma 后 Gray MAE 850→7222），快速回滚。这个 workflow 在我们有限时间内快速排除错误路径。

## 剩余未解问题

### 1. BW 58/59 FAIL（最大 cluster）

**证据**：
- 色卡 `c_bw_baseline_000` 在 G8 tile（input 34952 u16 gray）：ours=13388，ref=15424，差 2036 u16
- gamma=0.231（非默认 0.2）能消除 signed bias（-862 → -33），但 MAE 不变
- `FFF_BW_NO_HIGHSHADOW=1` 让 MAE 更差（3210），说明 HighShadow 现有扩展有用

**T56 ghidra agent 结论**：
- `FUN_702664e0` CGammaNegCurve builder **无 film_type 分支** — gamma 默认普世 0.2
- Mode=5 BW 路径无额外 curve
- Hasselblad Gray.icc 仅 profile name marker
- 0.231 推测来自 **scanner 注册表/config**（`HKCU\Software\Hasselblad\FlexColor` 或 scanner `.dfX:` profile 内嵌 gamma），DLL `.rdata` 内不可达

**下一 session 推进路径**：
- **方向 A**：Windows 机器上 `reg query HKCU\Software\Hasselblad\FlexColor` dump scanner 配置
- **方向 B**：深解 `profiles/Flextight X5 & 949.icc` 的 DevD/CIED/Pmtr 私有 tags（本次已 peek，未见 gamma 字段）
- **方向 C**：接受 BW 为 "structural limitation"，文档化

### 2. gamma_050 (3 case, FAIL 但 MAE 已大幅降)

**现状**：
- v_pos_gamma_050: MAE 5659 → 2039 (T57 改进后)
- v_neg_gamma_050: MAE 11121 → 7889
- 属 FlexColor UI 禁用域（G<0.75 denom 负），T57 hardcode exp=3

**遗留**：exp=3 在 G=0.5 是精确值（从 ref 反推），但对其他 G<0.76 值不精确。本 session 没有其他 G<0.76 测试点，无法校准完整曲线。

### 3. histo_gray_low (1 POS FAIL, MAE 1302)

**证据**：
- FFF XML 对比 baseline：唯一差异是 `<Gray>` 数组 `[128,128,128,128]` → `[96,96,96,96]`
- 但 **T29 + 本 session 独立 agent 都确认**：DLL 内 `+0x536` Gray 数组**无 pipeline 路径**（26 条引用全是 UI/XML save）
- 然而 ref TIF pixel 确实不同（R+1040, G-1132, B+360 at 1800,2244）

**推测**：FlexColor 批量导出时的状态缓存 artifact（AutoShadow 重算、dialog 状态污染等），非 pipeline bug。

**验证方案**：用户**单独**（非批量）重导 `v_pos_histo_gray_low.fff`，若 ref 等于 baseline 则确认是 batch-state 污染。

## 技术决策日志

### Lightness luma 公式（T53）
- **不是**：BT.601 (0.299R + 0.587G + 0.114B)
- **是**：算术均值 (R+G+B)/3
- **反编译证据**：`FUN_702d4720` 读取权重常量 `0.333f × 3 @ 0x70733f80`

### BW RGB→Gray collapse（T47）
- **不是**：lcms2 ICC transform via Hasselblad Gray.icc
- **是**：BT.601 整数 luma `(R*299 + G*587 + B*114) / 1000`（14-bit 域）
- **反编译证据**：`FUN_7025b1f0` 直接使用这个公式，权重 `float[3] = {0.299, 0.587, 0.114} @ 0x706e2864`

### USM gain（T55）
- **历史拟合**：67（rgb_standard/cmyk_standard 场景 R²=0.92）
- **当前**：50（色卡硬边 transition 匹配更好，amount=250 → gain=5.0 整数语义）
- 环境变量 `FFF_USM_GAIN_DIVISOR` 仍可覆盖

### Gamma G<0.76 fallback（T57）
- FlexColor UI 禁止 G<0.75，denom=1-(2-G)*0.8 为负
- **fallback 取 exp=3**（从 v_pos_gamma_050 ref 数值反推）
- 非 bit-accurate（仅 G=0.5 精确），但避免 LUT 全零灾难

### ICC tag 顺序
- 标准 TIFF ICC tag：**0x8773 (34675)**
- Imacon proprietary：0xC51A（FFF 内嵌）
- 历史 bug：`color::extract_embedded_icc` 只查 0xC51A。FFF 通常无 `acsp` 标记（确实无 ICC），回退到 bundled；但 **ref TIF 的 0x8773 完全被忽略**，导致短路未触发。
